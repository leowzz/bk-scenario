# 愿景与原则

## 愿景

构建一个面向后端开发与运维的“状态维护简化平台”：

- 用户在统一仪表盘中管理多个项目。
- 每个项目包含一组可点击的大按钮（规则入口）。
- 每个规则可由图形化节点编排定义复杂行为。
- 执行行为可覆盖 SQL、代码、日志、Shell、数据存取等动作。

## 要解决的问题

- 后端状态维护分散在脚本、SQL 片段、临时命令中，难以复用和审计。
- 跨项目操作缺少统一入口与隔离边界。
- 执行后的数据和上下文无法稳定回放。

## 产品原则

- 可视化优先：规则可在图形界面创建、编辑、运行。
- 项目隔离：默认按项目隔离变量、规则和执行数据。
- 可回溯：每次执行都要保留输入、步骤、产物和摘要。
- 可扩展：节点类型可新增，执行器保持插件化。
- 风险可控：执行高风险动作时必须有安全边界。

## 非目标

- 不作为通用 CI/CD 平台。
- 不替代完整工作流编排产品（如 Airflow/Temporal）。
- 不追求跨云跨区域强一致调度。
